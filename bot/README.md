## Telegram Bot (Guroo)

### Что делает
- Проверяет, что пользователь состоит в группе (если включено).
- Проверяет, зарегистрирован ли пользователь по `telegram_id`.
- Выдаёт ссылку на регистрацию тьютора через токен (ОДИН РАЗ).

### Настройка
1. Скопируй `.env.example` в `.env` и заполни:
   - `BOT_TOKEN` — токен бота
   - `BOT_USERNAME` — username бота
   - `BOT_GROUP_ID` — id телеграм‑группы (с минусом)
   - `API_BASE_URL` — адрес FastAPI (например `http://localhost:8001`)
   - `FRONTEND_BASE_URL` — адрес фронта (например `http://localhost:5173`)
   - `GROUP_INVITE_URL` — ссылка‑приглашение в группу (опционально)
   - `START_COOLDOWN_SECONDS` — антиспам задержка на команды
   - `API_TIMEOUT` / `API_RETRIES` — устойчивость API
   - `BOT_ADMIN_IDS` — id админов через запятую (для `/stats`)
   - `BOT_BACKEND_ADMIN_KEY` — ключ для админ‑операций в API (для `/reset_invite`)
   - `BOT_ENABLE_DEBUG_COMMANDS` — включить/выключить debug команды в проде

2. Установи зависимости:
   - `pip install -r requirements.txt`

### Запуск
`python main.py`

### Команды
- `/start` — получить ссылку регистрации
- `/link` — получить ссылку ещё раз
- `/checkme` — проверить статус в группе
- `/whoami` — показать ваш id
- `/ping` — проверка бота
- `/stats` — статистика (для админов)
- `/reset_invite <telegram_id|me>` — сбросить “память” о выданной ссылке (только админы)
- `/reset_invites_all` — полный сброс памяти у бота (только админы)

### Флоу (важно)
- Теперь бот делает проверку доступа вручную: `/start` показывает кнопку **Проверить доступ**.
- Пока пользователь не состоит в группе, ссылку он не получит.
- Если ссылка уже выдавалась ранее — бот второй раз её не выдаёт (защита от лже‑тьюторов).
